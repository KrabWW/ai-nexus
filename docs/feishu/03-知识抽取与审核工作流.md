# 03-知识抽取与审核工作流

> 来源：飞书知识库 | 最后编辑：2026-04-12

## 核心流程

```
commit message / PR 描述 / 开发日志
         │
         ▼
   AI 提取引擎（Knowledge Extractor）
         │
         ▼
   知识候选项（status: pending）
         │
         ▼
   人工审核界面（通过 / 修改 / 拒绝）
         │
    ┌────┼────┐
    │    │    │
 通过  修改 拒绝
    │    │    │
    ▼    ▼    ▼
 approved→入库  修改后重审  丢弃+记录
```

## 知识抽取

### 数据来源

- commit message（Claude Code hooks 自动触发）
- PR 描述
- 开发日志（logs/*.md）
- 飞书文档（可选接入）

### AI 提取逻辑

输入：一段文本（如 commit message）

```
feat: 新增ICU排班模块，支持24小时值班规则，排班周期7天
```

输出：结构化知识候选项

```json
{
  "entities": [
    {"name": "ICU排班", "type": "功能模块", "confidence": 0.95},
    {"name": "24小时值班", "type": "规则", "confidence": 0.98}
  ],
  "relations": [
    {"source": "ICU", "type": "需要", "target": "24小时值班", "confidence": 0.92}
  ],
  "rules": [
    {
      "name": "ICU 24h值班",
      "description": "ICU需要24小时值班医生",
      "domain": "排班",
      "severity": "error",
      "confidence": 0.98
    },
    {
      "name": "排班周期",
      "description": "排班周期为7天",
      "domain": "排班",
      "severity": "info",
      "confidence": 0.90
    }
  ]
}
```

### 提取 Prompt（核心）

```markdown
你是业务知识提取专家。从开发者文本中提取结构化业务知识。

提取目标：
1. 业务实体：识别业务名词，判断类型（人物/地点/机构/概念/系统）
2. 业务关系：识别实体间关系，判断方向和类型
3. 业务规则：识别约束条件，判断严重级别（error/warning/info）

每个提取项需要：
- 置信度评分（0-1）
- 归属业务领域

注意：
- 只提取业务知识，不提取技术实现细节
- 不提取通用的编程概念（如"数据库连接池"）
- 优先提取硬性约束（"必须""不能""至少"等关键词）
```

## 审核工作流

### 状态机

```
pending（AI提取/手动提交）
   │
   ├─▶ approved → 正式入库（entities/relations/rules 表）
   ├─▶ modified → 修改后重新进入 pending
   └─▶ rejected → 丢弃，记录到 knowledge_audit_log
```

### 审核界面（MVP）

最小化 Web 界面，三个操作：
- **通过**：直接入库，status → approved
- **修改**：编辑后重新提交，status → pending
- **拒绝**：丢弃并记录原因

### 审核记录

所有审核操作记录到 `knowledge_audit_log` 表，包含：
- 操作人、操作时间
- 变更前后快照（old_value / new_value）
- 操作类型（create/update/approve/reject）

## 触发方式

### 方式1：Claude Code Hook（自动）

post_task Hook 在 AI 完成任务后自动分析变更，提取知识候选项。

### 方式2：MCP 工具（手动）

通过 `submit_knowledge_candidate` 工具手动提交。

### 方式3：Git Hooks（自动）

- `commit-msg`：提交前自动校验 commit message 是否违反业务规则（warning only）
- `post-commit`：提交后自动调用 LLM 抽取知识，提交为待审核候选
- 安装方式：`bash scripts/install-hooks.sh`

### 方式4：冷启动引导

`POST /api/cold-start` 根据领域描述，LLM 生成初始知识框架（实体+规则+关系），提交为待审核候选。

### 方式5：定时任务（批量）

每日定时分析开发日志，批量提取。
