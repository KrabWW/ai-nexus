# AI Nexus - 项目总览与路线图（修订版）

> 来源：飞书知识库 | 最后编辑：2026-04-12

## 一句话定位

AI Nexus 是为**有历史包袱的企业研发团队**设计的业务知识治理层，把企业特有的业务规则和实体关系嵌入 AI 开发循环，让 AI 像资深员工一样懂业务。

## 与 OpenViking/mem0 的关系

| | OpenViking / mem0 | AI Nexus |
|---|---------|----------|
| **解决什么** | AI 不忘事（通用上下文检索） | AI 懂业务（结构化业务规则） |
| **本质** | 通用 Context 数据库 | 业务知识治理层 |
| **关系** | **上游基础设施，复用不竞争** | 下游业务层 |

分工明确：
- OpenViking 负责：背景文档存储、通用语义检索、L0/L1/L2 分层上下文压缩
- AI Nexus 负责：业务实体+关系+规则的结构化图谱、开发流程 Hook、知识审核工作流

## AI Nexus 做的三件事

### 1. 业务知识图谱（核心壁垒）

将企业特有的业务实体、关系、规则结构化成可查询的图谱。

- 实体管理（CRUD，带属性）
- 关系管理（有向关系，带权重/条件）
- 规则管理（结构化业务约束）
- 图谱可视化（D3.js）

### 2. 开发流程 Hook（第二壁垒）

在 Claude Code 关键节点自动注入业务上下文或做规则校验。

- pre_plan：根据任务关键词，从图谱检索相关实体和规则，注入为 AI 工作上下文
- pre_commit：对即将提交的代码变更，检查是否违反已知业务规则

### 3. 知识审核工作流（企业必需）

AI 从开发日志中自动抽取业务规则候选项，交由人工确认后才正式入图谱。

- 知识候选项自动生成（从 commit message、PR 描述、设计文档抽取）
- 待审核列表（Web 界面，通过/修改/拒绝）
- 审核通过后自动同步至图谱和 MCP 工具

## MVP 路线图

### Phase 0（2 周）— 基础设施接入

- 选定语义检索方案：代理 mem0 或直接图遍历
- 搭建 MCP Server 框架，跑通 Claude Code 接入
- 不写业务逻辑，只验证工具链
- **里程碑**：Claude Code 能通过 MCP 调用一个空工具并得到响应

### Phase 1（4 周）— 业务知识图谱 MVP

- 实体 + 关系 + 规则的 CRUD API
- search_entities / search_rules MCP 工具上线
- 最简单的管理 UI（列表 + 编辑，无需可视化）
- **里程碑**：Claude Code 能通过 MCP 查到一条真实业务规则

### Phase 2（3 周）— 开发流程 Hook

- pre_plan 注入业务上下文
- pre_commit 规则校验
- 与真实项目集成
- **里程碑**：有一个可演示的「AI 因业务规则被纠正」的 demo

### Phase 3（3 周）— 知识审核工作流 ✅ 已完成

- AI 从 commit message / PR 自动抽取候选知识
- 最小化审核界面（通过 / 拒绝）
- LLM 知识抽取引擎（Anthropic SDK，支持兼容 API）
- 冷启动引导（根据领域描述生成初始知识框架）
- Git Hooks 自动触发（commit-msg 校验 + post-commit 抽取）
- 知识飞轮闭环：commit → AI 抽取 → 人工审核 → 入库 → hook 可查
- **里程碑**：知识飞轮已闭环，Web Console 提供完整管理界面

## 不应该做的事（防止范围蔓延）

- 不要自建向量数据库层 — 语义检索代理 mem0，AI Nexus 只做图遍历
- 不要做通用 Agent 记忆管理 — 这是 OpenViking 的核心
- 不要把「知识库」做成文档库 — 文档库有 Confluence/飞书
- 不要过早做 AI Supervisor 复杂调度 — MVP 先把图谱 + Hook 做通
- 不要在 Phase 1 做完整可视化面板 — 先验证核心价值

## 技术栈

| 层 | 技术 |
|---|------|
| 后端 | Python 3.11+, FastAPI |
| 结构化存储 | SQLite |
| 语义检索 | mem0（内含向量数据库，AI Nexus 不直接管理） |
| AI 协议 | MCP (Model Context Protocol) |
| Hook 集成 | Claude Code hooks |
| 前端（后期） | React + D3.js |

## 系统架构

```
Claude Code / Cursor / Copilot
         │
         ▼ 同一 MCP 入口
    ┌────────┴────────┐
    │                 │
 AI Nexus          mem0/OpenViking
 实体图谱+规则引擎   通用记忆层+语义检索
    │                 │
 SQLite            mem0 内部向量库
 (图谱核心)        (AI Nexus 不直接管理)
```

AI Nexus 和 mem0/OpenViking 平级共存，各司其职：
- AI Nexus：业务规则（约束，不可违反）
- mem0：会话记忆（参考，可被覆盖）
